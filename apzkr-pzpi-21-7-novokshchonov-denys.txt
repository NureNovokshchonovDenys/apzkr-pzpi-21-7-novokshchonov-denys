Об’єктом дослідження є процес контролю якості повітря на основі даних, отриманих за допомогою датчиків. У сучасному світі проблема якості повітря має значну роль, тому для багатьох людей вимірювання характеристик є важливим. Процес включає в себе використання багатьох приладів. Процес викликає труднощі з затратами часу та зональним вимірюванням властивостей повітря.
Метою курсової роботи є розробка системи, яка збирає інформацію з датчиків якості повітря, розраховує середнє значення для приміщення, повідомляє про можливі загрози. Така система має забезпечувати зручний інтерфейс для швидкого контролю відповідно до заданих приміщень та датчиків, що дозволяє користувачам з легкістю отримувати портрібні данні.
Методи розробки базуються на технології React Native для мобільного додатку, Html та JavaScript для веб-інтерфейсу адміністрування, MongoDB для бази даних, та Arduino IDE для розробки IoT-пристрою. Для реалізації було створено наступні колекції бд: users (дані про користувачів), sensors (дані про сенсори), rooms (приміщення).
У результаті роботи здійснено програмну реалізацію системи для контролю якості повітря. Програмна система складається з веб-додатку, сервера, мобільного додатку та смарт-пристрою.

    1. БІЗНЕС-ВИМОГИ
        1.1. Передумови

У сучасному світі контроль якості повітря вимагає значних зусиль та часу від користувачів, які прагнуть забезпечити оптимальні умови для свого здоров'я та комфорту. Процес включає моніторинг показників з різних джерел, таких як температура, вологість, тиск та рівень забруднення, що може бути викликом, особливо коли дані важко доступні.

У цьому контексті, програмна система, яка забезпечує персоналізований моніторинг та спрощує процес отримання даних про якість повітря, стає ключовою для забезпечення зручності та ефективності. Перш за все, користувачі стикаються з проблемою аналізу великого обсягу даних, де важливо не лише враховувати поточні умови, але й розуміти можливі ризики для здоров'я. Система повинна надавати рекомендації, засновані на аналізі індивідуальних потреб користувача та актуальних показників якості повітря, забезпечуючи прийняття рішень, що відповідають їхнім потребам.

Крім того, питання зручності у доступі до даних також є важливим. Користувачам необхідно легко отримувати інформацію про якість повітря без необхідності витрачати багато часу на пошук даних. Така система має забезпечувати зручний інтерфейс для швидкого отримання даних відповідно до заданих параметрів, що дозволяє користувачам з легкістю контролювати умови в своєму середовищі.

Нарешті, персоналізація моніторингу відіграє ключову роль у задоволенні потреб сучасних користувачів. Система має надавати індивідуалізовані рекомендації та сповіщення, враховуючи особисті потреби та налаштування, що допомагає користувачам відчувати персоналізований підхід та забезпечує більш задовільний досвід моніторингу.

Таким чином, розробка програмної системи для персоналізованого моніторингу якості повітря та спрощення процесу отримання даних є важливою для користувачів, які шукають зручність, ефективність та персоналізацію у контролі якості повітря.

    5. СЕРВЕРНАЯ ЧАСТИНА ПРОДУКТУ
        5.1. База даних

У ході роботи було розроблено базу даних з використанням реляційної СУБД MongoDB.
MongoDB є широко використовуваною системою керування базами даних в документному форматі, зокрема має переваги як легке масштабування та простота взаємодії.
 Функції цієї системи дозволяють зручно . 
    • Python: MongoDB має PyMongo драйвер, що дозволяє з'єднуватися з базою даних MongoDB, виконувати запити та обробляти результати в Python.
Для реалізації було створено наступні таблиці:
        ◦ users: колекція з даними про клієнтів;
        ◦ sensors: колекція з даними датчиків;
        ◦ rooms: колекція з даними про приміщення;


        5.2. Backend

Для написання серверної частини коду в проекті, що стосується контролю якості повітря, використовувалися такі технології та бібліотеки:
    • Python: мова програмування, що часто використовується для розробки веб-додатків. Python може взаємодіяти з базою даних MongoDB, виконувати запити та обробляти результати.
    • MongoDB: це система керування базами даних. MongoDB забезпечує високу продуктивність, надійність і легкість у використанні. Функції цієї системи дозволяють легко масштабуватись.
    • Flask: це веб-сервер з відкритим кодом, який забезпечує надійність та гнучкість в розгортанні веб-додатків.

Для зв'язку з базою даних і визначення моделей у Python використовується об'єктно-орієнтований підхід. Були розроблені моделі, які представляють json-форматовані об’єкти та забезпечують зручний інтерфейс для взаємодії з ними.
Щодо шифрування паролів та створення токенів авторизації, завдяки Python я використав generate_password_hash з werkzeug.security.

Хешування паролів забезпечує безпеку, оскільки навіть у разі несанкціонованого доступу до бази даних, зловмисники не зможуть дізнатися справжні паролі користувачів. Замість цього вони побачать тільки хешовані значення, які неможливо легко зворотно перетворити у вихідні паролі.

        5.3. Структура проекту


Проект було структуровано на три шари:
    1. Інтерфейс	користувача	–	налаштування	та	запити, за допомогою яких
користувач взаємодіє з контролерами та маршрутами.
    2. Логіка бізнес-процесів – обробка та управління даними.
    3. Рівень доступу до даних – робота з базою даних, міграції та забезпечення коректної взаємодії бази даних із сервером.


        5.4. Cпецифікація

При написанні запитів було обрано архітектурний стиль REST. Основні принципи REST, які використовуються в проекті, включають
REST (Representational State Transfer) використовується для створення ефективних та масштабованих API. REST забезпечує надійний обмін даними між клієнтом і сервером, що є критично важливим для роботи системи.
Всі ресурси системи доступні через стандартизовані методи HTTP (GET, POST, PUT, DELETE). Це забезпечує прозорість та передбачуваність взаємодії між клієнтом і сервером. Наприклад, для отримання інформації про клієнта використовується метод GET, для створення нового клієнта - POST, для оновлення даних клієнта - PUT, а для видалення клієнта - DELETE.
